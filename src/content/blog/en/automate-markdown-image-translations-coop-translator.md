This was the first public marker of a shift in my work.

Before this, much of my writing was about building AI applications: RAG systems, fine-tuning workflows, Prompt Flow, and Azure setup. With the Phi-3 Cookbook case study, I started writing about the infrastructure around technical knowledge itself.

The challenge was not only translating Markdown. It was preserving structure, translating image text, organizing output folders, and making the process repeatable for a real open-source project.

That experience changed my direction. I began to see documentation as a living system, and translation as something that needed engineering discipline.
